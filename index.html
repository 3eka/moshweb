Mosh

SSH reinvented for the mobile world

Mosh roams from Ethernet to Wi-Fi to cell.

Mosh comes back quickly after a connection drop or resume from suspend.

Mosh has instant local editing that works with your favorite text editor or IRC client.

Mosh handles latency and packet loss better than SSH.

Mosh is free, open-source software, available for GNU/Linux and Mac OS X.

Getting Mosh »

Change IP, stay connected.

Mosh adjusts instantly as you move between Internet connections. Wi-Fi on the train, Ethernet in the hotel, 3G on the beach. You stay logged in, effortlessly.

Soar over Internet speedbumps.

Train entering a tunnel? Cosmic rays dazzled your cable modem? Just sit back and relax. Mosh is back online as soon as you are. Or suspend your laptop and return to a live Mosh session in a minute or a week.

What lag?

Mosh gives you lag-free editing by predicting the effects of your keystrokes before the server acknowledges them. Not just at the command line; it also works in full-screen curses programs like your text editor. Stale predictions are underlined so you aren't misled.

Secure by reduction.

Mosh connection setup happens over SSH, so it works with your SSH password, public key, or secret agent keyfob. You don't need special privileges to install or run Mosh. There's no persistent daemon or setuid nonsense. Just one process per connection, running as your user. And Mosh protects your connection with industry-standard authenticated encryption.

Your favorite GUI, or none at all.

Mosh is a command-line program, just like ssh. It works inside xterm, gnome-terminal, OS X's Terminal.app, or that green-screen VT220 gathering dust in your basement. But Mosh is designed from scratch for the modern world of UTF-8 Unicode, so it will actually compensate for Unicode bugs in other programs.

Smart protocol, instant ^C.

Mosh's UDP-based protocol has custom flow control which understands terminals and estimates network conditions. It recovers from dropped packets much faster than TCP-based SSH. And Mosh won't fill up network buffers, so you can kill a runaway process immediately with a single Control-C.

Getting Mosh

Debian GNU/Linux unstable

# apt-get install mosh

Ubuntu 10.04 LTS or later

$ sudo add-apt-repository ppa:keithw/mosh
$ sudo apt-get update
$ sudo apt-get install mosh

MacPorts OS X 10.6 or later

$ sudo port install mosh

Homebrew OS X 10.6 or later

$ brew install mobile-shell

Static server binary

If you cannot install a Mosh package on the server, you can put a statically-linked mosh-server binary in your home directory.

You will need to launch mosh as

$ mosh --server=./mosh-server
